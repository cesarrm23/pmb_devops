PatchMyByte DevOps - Manual de Usuario

Guía completa para administrar tus instancias Odoo desde una plataforma central.

1. Acceso al Módulo

1 Inicia sesión en asistentelisto.com

2 Haz clic en el menú de hamburguesa y selecciona DevOps

Verás el dashboard de Proyectos con los submenús: Proyectos, Git Graph, Ramas, Builds, Deploy IA, Logs, Backups, Terminal, Asistente IA, Plugins y Configuración.

2. Crear un Proyecto

1 Ve a DevOps > Proyectos y haz clic en Nuevo

2 Llena los campos del formulario:

CampoEjemplo (Cremara)Descripción
NombreCremaraNombre del negocio/proyecto
Ruta del Repositorio/opt/odoo19Test/cremara_addonsRuta absoluta al repo Git
Servicio systemdodoo19TestNombre del servicio (sin .service)
URLhttps://test.cremara.comURL de la instancia
Base de Datosodoo19testNombre de la BD PostgreSQL
EntornoStagingProducción/Staging/Desarrollo
ConexiónLocalLocal (mismo servidor) o SSH

3 En la pestaña Miembros, agrega usuarios y asigna roles (Admin/Developer/Viewer)

4 Guarda el proyecto

3. Sincronizar Ramas

1 Abre un proyecto y haz clic en Sincronizar Ramas

Las ramas se clasifican automáticamente como Production, Staging o Development según la configuración del proyecto.

Las ramas se pueden ver también en el Git Graph con visualización por colores.

4. Terminal Web

La terminal tiene 3 pestañas:

TabFunción
AIClaude Code CLI — asistente IA con acceso al código
ShellBash interactivo en el servidor
Logsjournalctl en tiempo real del servicio

1 Selecciona un proyecto del dropdown

2 Haz clic en el tab deseado (AI/Shell/Logs)

5. Git Graph

Visualización de ramas con sidebar por tipo:

  • PRODUCTION — rama principal (main)
  • STAGING — rama de pruebas
  • DEVELOPMENT — ramas de desarrollo

6. Backups

1 Ve a DevOps > Backups para ver todos los backups

2 Para crear uno manualmente, abre un proyecto y haz clic en Crear Backup

Los backups automáticos se ejecutan diariamente vía cron. Configurar retención en Configuración > Ajustes.

7. Builds & Deploy

Build manual: Desde una rama, haz clic en Build (pull + restart)

Deploy con IA: Ve a Deploy IA y crea un nuevo deploy. El flujo es:

  1. Tests IA — Claude evalúa si es seguro deployar
  2. Backup automático de la BD
  3. Git pull de la rama origen
  4. Restart del servicio (nunca stop)
  5. Verificación HTTP
  6. Auto-rollback si algo falla
Importante: Solo administradores pueden deployar a producción.

8. Roles y Permisos

RolAcceso
SuperadminAcceso total a todos los proyectos. Deploy a producción en cualquier proyecto.
Admin (por proyecto)Deploy a producción en su proyecto. Gestión completa.
DeveloperTrabaja en staging/development. Terminal, AI, logs. NO deploy a producción.
ViewerSolo lectura. Ve branches, builds, logs, backups.

9. Configuración

Ve a DevOps > Configuración > Ajustes para configurar:

  • Claude Code: API Key, modelo IA, test de conexión
  • Backups: Ruta de almacenamiento, días de retención, auto-backup
  • Git: Ruta por defecto del repositorio
